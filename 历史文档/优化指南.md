### P2-3: CSRF防护（1小时）⭐⭐⭐

**效果**: 防止跨站请求伪造攻击

#### 步骤1: 后端配置（30分钟）

```java
// springboot/src/main/java/com/example/config/SecurityConfig.java
@Configuration
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf()
                .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
                .and()
            .authorizeHttpRequests()
                .requestMatchers("/user/login", "/user/register").permitAll()
                .anyRequest().authenticated();
        
        return http.build();
    }
}
```

#### 步骤2: 前端配置（30分钟）

```javascript
// vue-frontend/src/api/request.js
import axios from 'axios'

const request = axios.create({
  baseURL: 'http://localhost:19090',
  timeout: 10000,
  withCredentials: true // 允许携带Cookie
})

// 请求拦截器
request.interceptors.request.use(config => {
  // 从Cookie中读取CSRF Token
  const csrfToken = document.cookie
    .split('; ')
    .find(row => row.startsWith('XSRF-TOKEN='))
    ?.split('=')[1]
  
  if (csrfToken) {
    config.headers['X-XSRF-TOKEN'] = csrfToken
  }
  
  return config
})
```

### P0-4: 密码明文传输（1小时）⭐⭐⭐

**说明**: 对于本地毕设项目，当前BCrypt加密已足够安全

#### 方案1: 配置HTTPS（推荐生产环境）

```nginx
# nginx.conf
server {
    listen 443 ssl;
    server_name localhost;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://localhost:5173;
    }
    
    location /api {
        proxy_pass http://localhost:19090;
    }
}

# 强制HTTP跳转HTTPS
server {
    listen 80;
    server_name localhost;
    return 301 https://$server_name$request_uri;
}
```

#### 方案2: 前端加密（额外安全层）

**注意**: 这需要修改数据库中所有现有密码，**不推荐**

```bash
npm install crypto-js
```

```javascript

import CryptoJS from 'crypto-js'

const handleSubmit = async () => {
  const encryptedForm = {
```

```java
// 后端也需要相应修改
```

**结论**: 对于毕设项目，**不建议修复此问题**，因为：

## 📝 注意事项

### 1. 图片懒加载

- ✅ 简单易用，效果明显
- ⚠️ 需要修改所有图片标签
- ⚠️ 注意loading和error图片路径

### 2. 表单防抖

- ✅ 减少服务器压力
- ⚠️ 注意防抖时间（推荐300ms）
- ⚠️ 不要对所有输入都防抖

### 3. 虚拟滚动

- ✅ 性能提升巨大
- ⚠️ 需要固定item高度
- ⚠️ 可能影响现有布局

### 4. CSRF防护

- ✅ 提升安全性
- ⚠️ 需要前后端同时修改
- ⚠️ 可能影响现有接口

### 5. HTTPS配置

- ✅ 传输加密
- ⚠️ 需要SSL证书
- ⚠️ 本地开发可以使用自签名证书
