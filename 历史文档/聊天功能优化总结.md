## 🎯 优化 1：聊天未读数优化

### 问题描述

**修复前**：

- ❌ 顶部"消息"旁边的数字固定显示联系人总数
- ❌ 点击进入聊天后，未读数不会减少
- ❌ 无法区分有无未读消息

**修复后**：

- ✅ 顶部数字显示**总未读数**
- ✅ 有未读时：红色高亮 + 动画
- ✅ 无未读时：灰色显示联系人总数
- ✅ 点击进入聊天：自动清除该联系人的未读数
- ✅ 实时更新：总未读数动态计算

### 技术实现

**1. 计算总未读数**：

```javascript
const totalUnreadCount = computed(() => {
  return contacts.value.reduce(
    (sum, contact) => sum + (contact.unreadCount || 0),
    0
  );
});
```

**2. 点击清除未读数**：

```javascript
const selectContact = async (contact) => {
  activeChat.value = contact;
  showChatWindow.value = true;

  if (contact.unreadCount > 0) {
    contact.unreadCount = 0;
    wsStore.chatUnreadCount = totalUnreadCount.value;
  }

  // 加载聊天记录...
};
```

**3. 动态显示**：

```vue
<!-- 有未读：红色高亮 + 动画 -->
<span
  v-if="totalUnreadCount > 0"
  class="bg-gradient-to-r from-red-500 to-pink-500 text-white animate-pulse"
>
  {{ totalUnreadCount }}
</span>

<!-- 无未读：灰色 -->
<span v-else class="bg-gray-100 text-gray-500">
  {{ contacts.length }}
</span>
```

## 💡 技术亮点

### 1. 响应式计算

```javascript
const totalUnreadCount = computed(() => {
  return contacts.value.reduce(
    (sum, contact) => sum + (contact.unreadCount || 0),
    0
  );
});
```

**优势**：

- ✅ 自动更新
- ✅ 性能优化
- ✅ 代码简洁

### 2. 消息类型分离

```javascript
const unreadCount = ref(0); // 系统通知
const chatUnreadCount = ref(0); // 私信

const handleMessage = (msg) => {
  if (msg.type === "NOTICE") {
    unreadCount.value++;
  } else if (msg.type === "CHAT") {
    chatUnreadCount.value++;
  }
};
```

**优势**：

- ✅ 清晰的消息分类
- ✅ 独立的未读数管理
- ✅ 更好的用户体验

### 3. 视觉层次优化

**背景层次**：

1. 基础背景：from-slate-50
2. 装饰球：emerald-200/60
3. 网格纹理：linear-gradient
4. 内容层：白色卡片

**阴影层次**：

1. 联系人列表：shadow-xl
2. 消息气泡：shadow-lg
3. 悬停效果：shadow-md
