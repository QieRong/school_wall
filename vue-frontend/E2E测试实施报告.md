## 最佳实践

### 1. 测试独立性
每个测试用例应该独立运行，不依赖其他测试的状态。

### 2. 等待策略
使用 `waitForLoadState('networkidle')` 等待网络请求完成，避免不稳定的测试。

### 3. 选择器优先级
1. 文本选择器: `text=登录`
2. 占位符选择器: `input[placeholder*="学号"]`
3. 类名选择器: `.post-card`
4. data-testid: `[data-testid="submit-button"]`

### 4. 断言清晰
使用明确的断言，提供清晰的错误信息：
```javascript
await expect(page.locator('text=登录成功')).toBeVisible()
```

### 5. 数据清理
测试后清理创建的数据，避免污染数据库。

## 故障排查

### 常见问题

#### 1. 测试超时
**原因**: 网络慢、后端响应慢
**解决**: 
- 增加timeout配置
- 检查后端服务状态
- 优化网络环境

#### 2. 元素未找到
**原因**: 选择器错误、页面未加载完成
**解决**:
- 检查选择器是否正确
- 增加 `waitForSelector`
- 使用更可靠的选择器

#### 3. 测试不稳定
**原因**: 竞态条件、动画未完成
**解决**:
- 增加适当的 `waitForTimeout`
- 等待动画完成
- 使用 `waitForLoadState`

#### 4. 并发问题
**原因**: 多个测试同时操作同一资源
**解决**:
- 减少worker数量
- 使用不同的测试账号
- 添加资源锁

  test:

    steps:
      - uses: actions/checkout@v3

        run: |
          cd vue-frontend

        if: always()

          name: playwright-report
