# E2E测试新手指南 - 从零开始

## 什么是E2E测试？

**E2E = End-to-End（端到端）测试**

简单来说，就是让电脑自动打开浏览器，像真实用户一样操作你的网站：
- 自动点击按钮
- 自动填写表单
- 自动检查页面内容
- 自动验证功能是否正常

**好处**：
- 不用手动测试每个功能
- 几分钟就能测试完所有功能
- 答辩时可以演示自动化测试（加分项）

---

## 第一步：安装Playwright

Playwright是微软开发的自动化测试工具，非常强大。

### 1.1 打开命令行

**Windows系统**：
- 按`Win + R`
- 输入`cmd`
- 回车

### 1.2 进入前端项目目录

```bash
cd D:\桌面\wall\vue-frontend
```

（根据你的实际路径修改）

### 1.3 安装Playwright浏览器

```bash
npx playwright install --with-deps
```

**说明**：
- 这个命令会下载Chrome浏览器（用于测试）
- 大约需要5-10分钟（取决于网速）
- 只需要安装一次

**如果下载很慢**，可以设置国内镜像：
```bash
set PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright/
npx playwright install --with-deps
```

## 第二步：启动后端服务

E2E测试需要后端API正常运行。

### 2.1 打开新的命令行窗口

再打开一个cmd窗口（不要关闭之前的）

### 2.2 启动后端

```bash
cd D:\桌面\wall\springboot
mvn spring-boot:run
```

**等待看到这样的提示**：
```
Started SpringbootApplication in X.XXX seconds
```

**注意**：
- 后端必须运行在 `http://localhost:19090`
- 不要关闭这个窗口

### 3.1 再打开一个新的命令行窗口

现在你应该有3个cmd窗口了

### 3.2 启动前端


## 第四步：运行E2E测试

现在后端和前端都在运行，可以开始测试了！

### 方式一：UI模式（推荐新手）

**最简单、最直观的方式**

```bash
cd D:\桌面\wall\vue-frontend
npm run test:e2e:ui
```

**会发生什么**：
1. 自动打开一个测试界面
2. 左侧显示所有测试文件
3. 点击任意测试文件
4. 右侧会打开浏览器，自动执行测试
5. 你可以看到浏览器自动操作的全过程

**操作步骤**：
1. 点击左侧的`01-auth.spec.js`（用户认证测试）
2. 点击绿色的播放按钮▶️
3. 观看浏览器自动登录、注册
4. 测试通过会显示绿色✓

### 方式二：命令行模式（快速运行）

**适合运行所有测试**

```bash
cd D:\桌面\wall\vue-frontend
npm run test:e2e
```

**会发生什么**：
1. 自动运行所有93个测试
2. 在后台执行（看不到浏览器）
3. 4-6分钟后显示测试结果
4. 生成HTML报告

### 方式三：有头模式（看到浏览器）

**想看浏览器操作过程**

```bash
cd D:\桌面\wall\vue-frontend
npm run test:e2e:headed
```

**会发生什么**：
1. 打开真实的Chrome浏览器
2. 自动执行所有测试
3. 你可以看到浏览器自动操作
4. 速度比UI模式快

## 常见问题解决

### 问题1：端口被占用

**错误提示**：
```
Port 5173 is already in use
```

**解决方法**：
1. 关闭之前运行的前端服务
2. 或者修改端口号

### 问题2：后端没启动

**错误提示**：
```
connect ECONNREFUSED 127.0.0.1:19090
```

**解决方法**：
1. 检查后端是否正常运行
2. 访问 `http://localhost:19090` 确认

### 问题3：数据库连接失败

**错误提示**：
```
Could not connect to MySQL
```

**解决方法**：
1. 启动MySQL服务
2. 检查数据库配置（用户名、密码）

### 问题4：测试超时

**错误提示**：
```
Test timeout of 30000ms exceeded
```

**解决方法**：
1. 网络太慢，增加超时时间
2. 或者重新运行测试
