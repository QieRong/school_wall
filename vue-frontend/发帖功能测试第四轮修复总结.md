# 发帖功能测试第四轮修复总结

## 关键发现

### 1. Toast组件渲染结构
```vue
<!-- App.vue -->
<div class="fixed top-24 left-1/2 -translate-x-1/2 z-[9999] ...">
  <div v-for="msg in appStore.toasts" :key="msg.id" class="...">
    <CheckCircle2 v-if="msg.type === 'success'" class="w-5 h-5" />
    <span>{{ msg.text }}</span>  <!-- 纯文本渲染 -->
  </div>
</div>
```

**结论**：Toast必须使用`.fixed:has-text()`选择器定位

### 2. 发布按钮文本
根据用户提供的截图，发布按钮文本是：
```
发 布  （中间有空格）
```

**解决方案**：使用正则表达式`/发\s*布/`匹配

### 3. 收藏按钮DOM结构
```vue
<span class="flex items-center gap-2 px-3 py-1.5 rounded-full ...">
  <Star class="w-4 h-4" />
  <span class="font-medium">收藏</span>  <!-- 嵌套的span -->
</span>
```

**问题**：`span:has-text("收藏")`匹配到2个元素（外层和内层）
**解决方案**：使用`span.flex.items-center.gap-2.filter({ hasText: '收藏' })`

## 修复对比

| 测试 | 原选择器 | 新选择器 | 原因 |
|-----|---------|---------|------|
| 02-01~03 | `text=/发布成功/` | `.fixed:has-text("发布成功")` | Toast在fixed容器中 |
| 02-01~03 | `button:has-text("发 布")` | `button.filter({ hasText: /发\s*布/ })` | 按钮文本有空格 |
| 02-06 | `text=/链接已复制/` | `.fixed:has-text("链接已复制")` | Toast在fixed容器中 |
| 02-07 | `span:has-text("收藏")` | `span.flex.items-center.gap-2.filter({ hasText: '收藏' })` | 避免匹配多个元素 |

## 选择器最佳实践

### Toast验证
```javascript
// ❌ 错误
await page.waitForSelector('text=/发布成功/')

// ✅ 正确
await page.waitForSelector('.fixed:has-text("发布成功")')
```

### 按钮定位
```javascript
// ❌ 错误：精确匹配可能因空格失败
button:has-text("发 布")

// ✅ 正确：正则匹配兼容空格
button.filter({ hasText: /发\s*布/ })
```

### 避免Strict Mode
```javascript
// ❌ 错误：匹配到多个元素
span:has-text("收藏")

// ✅ 正确：使用更精确的class选择器
span.flex.items-center.gap-2.filter({ hasText: '收藏' })
```
