# 退出登录逻辑修复报告

## 🔴 问题描述

**测试用例**：`01-06: 用户退出登录`

**错误信息**：
```
Error: expect(page).toHaveURL(expected) failed
Expected: "http://localhost:3000/login"
Received: "http://localhost:3000/home"
Timeout: 5000ms
```

**根本原因**：退出登录后，系统没有跳转到登录页，而是停留在首页或刷新当前页面。

## 🔍 问题分析

### 发现的3个退出登录实现不一致问题

#### 1. Home.vue（首页）- 🔴 严重错误
**原代码**：
```javascript
const confirmLogout = () => { 
  userStore.logout(); 
  location.reload()  // ❌ 错误：刷新当前页面，不跳转
}
```

**问题**：
- 使用`location.reload()`刷新当前页面（`/home`）
- 刷新后虽然清除了登录状态，但仍停留在`/home`
- 路由守卫未正确重定向到登录页

#### 2. AdminLayout.vue（管理后台）- 🟡 不完整
**原代码**：
```javascript
const confirmLogout = () => {
  loggingOut.value = true
  setTimeout(() => {
    localStorage.removeItem('user')  // ❌ 手动清除，未调用store方法
    router.push('/login')
  }, 500)
}
```

**问题**：
- 只清除了localStorage，未调用`userStore.logout()`
- 导致WebSocket连接未断开
- 可能造成内存泄漏和连接残留

#### 3. UserLayout.vue（用户布局）- 🟡 调用不存在的方法
**原代码**：
```javascript
const handleLogout = () => {
  localStorage.removeItem('user')
  localStorage.removeItem('token')
  userStore.clearUser()  // ❌ 错误：clearUser方法不存在
  appStore.showToast('已退出登录', 'success')
  router.push('/login')
}
```

**问题**：
- 调用了不存在的`userStore.clearUser()`方法
- 应该使用`userStore.logout()`
- 手动清除localStorage是冗余操作

## ✅ 修复方案

### 统一退出登录标准流程

所有退出登录操作应遵循以下标准：

1. **调用`userStore.logout()`** - 统一的退出方法
   - 清除用户状态
   - 清除localStorage
   - 断开WebSocket连接

2. **使用`router.push('/login')`** - 跳转到登录页
   - 不使用`location.reload()`
   - 不手动操作localStorage

3. **可选：显示提示信息**
   - 使用`appStore.showToast()`

### 修复后的代码

#### 1. Home.vue
```javascript
const confirmLogout = () => { 
  userStore.logout()
  router.push('/login')
}
```

#### 2. AdminLayout.vue
```javascript
// 添加import
import { useUserStore } from '@/stores/user'
const userStore = useUserStore()

// 修复logout方法
const confirmLogout = () => {
  loggingOut.value = true
  setTimeout(() => {
    userStore.logout()
    router.push('/login')
  }, 500)
}
```

#### 3. UserLayout.vue
```javascript
const handleLogout = () => {
  userStore.logout()
  appStore.showToast('已退出登录', 'success')
  router.push('/login')
}
```

## 💡 最佳实践建议

### 1. 统一退出登录工具函数
建议创建`utils/auth.js`：

```javascript
import { useUserStore } from '@/stores/user'
import { useAppStore } from '@/stores/app'
import { useRouter } from 'vue-router'

export function useLogout() {
  const userStore = useUserStore()
  const appStore = useAppStore()
  const router = useRouter()
  
  return () => {
    userStore.logout()
    appStore.showToast('已退出登录', 'success')
    router.push('/login')
  }
}
```

### 2. 路由守卫增强
在`router/index.js`中确保未登录用户自动重定向：

```javascript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  if (!userStore.isLogged && to.path !== '/login') {
    next('/login')
  } else {
    next()
  }
})
```
